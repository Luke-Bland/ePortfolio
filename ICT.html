Video Analysis

While writing a practical report in physics, I relied on the use of a range of programs to gather, create and present data. The purpose of the practical was to find the effect of a projectiles cross sectional area on its horizontal range when launched perpendicular to the ground at a given height and velocity. It was difficult to gather much data without the aid of a video analysis program. Therefore, to determine the exact velocity, angle and motion of the projectiles, videos of the experiments were uploaded to a program called ‘Tracker.’ In order for it to work, the specific details of the setup must be entered. This includes one or more points of reference that provides a scale for Tracker to work with. It also requires axes to provide a frame of reference. Once this is done it is possible to make Tracker follow the motion of the projectile. Using the information provided, Tracker is able to calculate the values of the variables mentioned previously. With this new data, it was possible to calculate the approximate drag experienced by each projectile. Therefore, a correlation between the size of the projectile and the drag it would experience would be determined. Creating graphs on this data showed this correlation in a way that was easier to interpret.

ePortfolio

The creation of this ePortfolio relies heavily on the use of ICT. Before working on it, I had no experience in creating a webpage. Therefore, I relied on the help of others to point me in the right direction. I first had to learn how create the webpage itself. I found out that you could use GitHub to host the webpage. I was then able to create the homepage by using premade template, however, the layout did not fit the ePortfolio. In order to fix this I needed to change the way it was originally coded. To do this I needed to actually understand the coding. I learned that it was constructed from HTML (hypertext mark-up language). Using a website called Codecademy, I was able learn the basics required to insert text, rearrange structure, and create simple functions. This allowed me to create text boxes which could be clicked on to send the viewer to appendices. I made an appendix for each capability. Therefore, the information contained in the ePortfolio is easier to find and access. However, these while creating these appendices, I quickly realised that HTML relies on the assistance of CSS (cascading style sheets) to create a visually appealing design. I began to look into CSS so that I could improve the design, however, CSS proved to be much more complicated than HTML. Therefore, I had decided that it was not worth the time it would take to learn as it meant that I would have less time to input the actual information featured in the webpage.

OFC Election Math Folio

In Order From Chaos, we were required to predict the winner of the 2018 state election. In order to do this, I focused on the King electorate as it was the most marginal seat. It was a new district so I had to look at previous electoral outcomes in the area along with its demographics to predict whether the majority will vote Labor or Liberal. During the folio, I have become adept at using a range of functions in excel. I am able to collect and sort data in a meaningful and useable way. Functions that I have learned include calculating; mean/median/mode, midpoint, standard deviation, and variance of ranged data sets. Evidence of my learning can be found here. I proceeded to compare the data I found with two other sets I created showing the demographics of strong Liberal and strong Labor electorates. This allowed me to create justified and logical arguments in order to support my prediction.
